# SMBTunnel 功能需求文档

## 1. 概述

### 1.1 目标
开发一个名为 SMBTunnel 的工具，利用 SMB 协议在本地局域网中建立 TCP/UDP 隧道，使两台计算机能够通过 SMB 端口进行 TCP 连接。该工具旨在绕过可能存在的网络限制，确保在受限制的网络环境中能够进行可靠的通信。

### 1.2 背景
在某些网络环境中，TCP 和 UDP 端口可能受到严格限制，导致无法直接建立连接。通过利用 SMB 协议，SMBTunnel 可以绕过这些限制，提供稳定的通信通道。

### 1.3 目标用户
- 网络管理员
- 开发人员
- 需要在受限制网络环境中进行通信的用户

## 2. 功能需求

### 2.1 基本功能
1. **TCP 隧道建立**：
   - 通过 SMB 协议在两台计算机之间建立 TCP 隧道。
   - 支持双向数据传输。

2. **UDP 隧道建立**：
   - 通过 SMB 协议在两台计算机之间建立 UDP 隧道。
   - 支持双向数据传输。

3. **身份验证**：
   - 支持基于用户名和密码的身份验证。
   - 支持集成 Windows Active Directory 认证。

4. **数据加密**：
   - 支持数据加密，确保传输过程中的数据安全。
   - 支持 AES-256 加密算法。

5. **日志记录**：
   - 提供详细的日志记录功能，便于调试和监控。
   - 支持日志级别设置（如 DEBUG, INFO, WARNING, ERROR）。

6. **配置文件**：
   - 支持通过配置文件进行设置，便于管理和维护。
   - 配置文件格式为 YAML 或 JSON。

### 2.2 扩展功能
1. **压缩传输**：
   - 支持数据压缩，减少传输的数据量。
   - 支持 GZIP 和 ZLIB 压缩算法。

2. **流量监控**：
   - 提供实时流量监控功能，显示当前传输速率和总传输量。
   - 支持图表展示。

3. **多隧道支持**：
   - 支持同时建立多个隧道，每个隧道独立配置。
   - 支持隧道之间的隔离和管理。

4. **自动重连**：
   - 在网络中断后自动重新建立连接。
   - 支持重连间隔和最大重试次数设置。

5. **用户界面**：
   - 提供图形化用户界面（GUI），便于非技术用户使用。
   - GUI 支持隧道状态监控和配置管理。

## 3. 技术要求

### 3.1 硬件要求
- 支持主流的 x86 和 ARM 架构。
- 至少 1GB 内存。
- 至少 100MB 磁盘空间。

### 3.2 软件要求
- 操作系统：Windows, Linux, macOS。
- 编程语言：Go 1.18 或更高版本。
- 开发工具：Visual Studio Code, IntelliJ IDEA, 或其他支持 Go 的 IDE。
- 库和框架：
  - `golang.org/x/crypto`：用于加密和解密。
  - `github.com/go-smb/smb`：用于 SMB 协议的实现。
  - `github.com/spf13/viper`：用于配置文件解析。
  - `github.com/sirupsen/logrus`：用于日志记录。

### 3.3 安全要求
- 所有传输的数据必须经过加密处理。
- 支持强密码策略，防止弱密码攻击。
- 防止中间人攻击（MITM），确保数据传输的安全性。

### 3.4 性能要求
- 最小延迟：小于 100ms。
- 最大吞吐量：至少 100Mbps。
- 支持并发连接数：至少 100 个。

### 3.5 兼容性要求
- 支持 SMB 2.0 及以上版本。
- 与主流的操作系统和网络环境兼容。

## 4. 程序架构框架

### 4.1 模块划分
1. **主模块**：
   - 负责程序的启动、初始化和配置加载。
   - 处理命令行参数和配置文件。

2. **SMB 模块**：
   - 实现 SMB 协议的客户端和服务器功能。
   - 处理 SMB 数据包的封装和解封装。
   - 支持身份验证和会话管理。

3. **隧道模块**：
   - 实现 TCP 和 UDP 隧道的建立和数据传输。
   - 支持数据的加密和解密。
   - 处理数据包的转发和路由。

4. **加密模块**：
   - 提供数据加密和解密功能。
   - 支持 AES-256 加密算法。
   - 处理密钥生成和管理。

5. **日志模块**：
   - 提供日志记录功能。
   - 支持日志级别设置。
   - 支持日志文件的滚动和归档。

6. **配置模块**：
   - 解析和加载配置文件。
   - 支持动态配置更新。
   - 提供默认配置选项。

7. **用户界面模块**（可选）：
   - 提供图形化用户界面。
   - 显示隧道状态和配置信息。
   - 支持用户交互和配置管理。

### 4.3 接口定义

#### 4.3.1 SMB 模块接口
- `Connect(server string, username string, password string) error`：连接到 SMB 服务器。
- `Disconnect() error`：断开与 SMB 服务器的连接。
- `Send(data []byte) error`：发送数据到 SMB 服务器。
- `Receive() ([]byte, error)`：从 SMB 服务器接收数据。

#### 4.3.2 隧道模块接口
- `StartTCP(localAddr string, remoteAddr string) error`：启动 TCP 隧道。
- `StartUDP(localAddr string, remoteAddr string) error`：启动 UDP 隧道。
- `Stop() error`：停止隧道。

#### 4.3.3 加密模块接口
- `Encrypt(data []byte) ([]byte, error)`：加密数据。
- `Decrypt(data []byte) ([]byte, error)`：解密数据。

#### 4.3.4 日志模块接口
- `Log(level LogLevel, message string)`：记录日志。
- `SetLogLevel(level LogLevel)`：设置日志级别。

#### 4.3.5 配置模块接口
- `LoadConfig(filename string) error`：加载配置文件。
- `GetConfig(key string) interface{}`：获取配置项。
- `SetConfig(key string, value interface{})`：设置配置项。

### 4.4 数据模型

#### 4.4.1 配置文件示例（YAML）
```yaml
server:
  address: "192.168.1.100"
  port: 445
  username: "admin"
  password: "password"

tunnel:
  type: "tcp"  # 可选值: "tcp", "udp"
  local_address: "127.0.0.1:1080"
  remote_address: "192.168.1.200:80"

encryption:
  enabled: true
  algorithm: "aes-256"

logging:
  level: "info"
  file: "smbtunnel.log"
```
